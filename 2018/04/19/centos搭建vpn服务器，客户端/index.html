用centos虚拟机搭建VPN服务器,客户端

必备技能

项目需要，用搭建三个centos虚拟机，中一个做服务器，另两个为客户端，使用的pptpd搭建，成功实现连接。

步骤如下：

一.在vmware上安装三个centos虚拟机，我用的6.8版本，安装 很简单，网上有教程

二.在centos（1）上搭建pptpd做vpn服务器

  • 1.) 安装ppp
    打开命令提示符，进入root用户
    输入命令：
    yum install ppp

  • 2.）安装pptpd
    输入命令：
    yum update
    yum install pptpd

  • 2.)rpm下载安装
    wget ftp://ftp.ntua.gr/pub/net/poptop/pptpd/pptpd-1.3.1/pptpd-1.1.3-4.i386.rpm
    注意32位或者64位版本，否则吃大亏！鄙人就是在64位服务器上装了32位的pptpd，就给搞了很长时间才发现！！！！

  • 3.）配置你的pppd和pptpd

    • 1.配置/etc/pptpd.conf
      输入输入命令：
      vi /etc/pptpd.conf
      如图

      localip 192.168.1.1
      remoteip 192.168.1.2-238
      其中localip是VPN连接成功后，VPN server（就是你启动pptpd服务）的地址
      remoteip是是 VPN 拨号过来后分配给客户端的地址

    • 2.配置/etc/ppp/options.pptpd
      输入命令：
      vi /etc/ppp/options.pptpd
      如图：

      更改了
      ms-dns 8.8.8.8
      ms-dns 8.8.4.4

    • 3.编辑 /etc/ppp/chap-secrets
      vi /etc/ppp/chap-secrets
      如图：

      添加用户，如zxb1为其中一个用户，123456为密码，*不能掉

    • 4.配置文件/etc/sysctl.conf
      vim /etc/sysctl.conf
      修改以下内容开启ip转发：
      net.ipv4.ip_forward = 1
      保存、退出后执行：
      sysctl -p

    • 5.打开防火墙端口
      将Linux服务器的1723端口和47端口打开，并打开GRE协议。
      iptables -D INPUT -j REJECT –reject-with icmp-host-prohibited
      iptables -D FORWARD -j REJECT –reject-with icmp-host-prohibited
      iptables -A INPUT -i lo -p all -j ACCEPT
      iptables -A INPUT -p gre -j ACCEPT
      iptables -A INPUT -p tcp -m tcp –dport 21 -j ACCEPT
      iptables -A INPUT -p tcp -m tcp –dport 22 -j ACCEPT
      iptables -A INPUT -p tcp -m tcp –dport 47 -j ACCEPT
      iptables -A INPUT -p tcp -m tcp –dport 80 -j ACCEPT
      iptables -A INPUT -p tcp -m state –state NEW -m tcp –dport 1723 -j ACCEPT
      iptables -A FORWARD -p tcp –syn -s 192.168.0.0/24 -j TCPMSS –set-mss 1356
      iptables -t nat -A POSTROUTING -s 192.168.0.0/24 -o eth0 -j MASQUERADE

      这个最蛋疼，开始没注意，能连接上怎么都打不开网页，搞了半天才发现DNS端口没有打开，差点昏死过去！！
      service iptables save

      设置开机自启动
      chkconfig pptpd on
      chkconfig iptables on

  • 4.）检测
    打开防火墙，打开服务器
    输入命令：

    service iptables restart
    service pptpd start

三.在centos上连接pptpd服务器vpn客户端配置

  • 1.）安装pptp和pptp-setup
    yum install pptp pptp-setup

  • 2.）创建配置
    pptpsetup –create test –server 192.168.19.65 –username zxb1 –password 1234546 –encrypt
    其中test为配置名随意起，192.168.19.65服务器的ip地址（vpn转接前的），zxb1是服务器分配的用户之一，123456为该用户密码。

  • 3.）连接vpn
    pppd call test

  • 4.)验证连接
    ip a |grep ppp
    如果此时没有丝毫反应，就说明连接失败，解决这个问题花了我好久，因为我用pptpsetup –create test –server 192.168.19.65 –username zxb1 –password 1234546 –encrypt –start这个命令时连接不成功，但是却没有报错，我只是感觉有可能是连接超时造成，在goole上用别人的办法终于解决了，是因为centos缺少 modprobe模块
    在2.）之前输入命令
    sudo modprobe nf_conntrack_pptp

  • 5.)查看连接情况
    输入 route -n
    会看到网络中多了一个ppp0连接
    恭喜你已经连接成功

    不过别高兴早了，在浏览器中输入IP11.cn查看网络情况，会发现网络根本没有经过vpn服务器，还是之前的网络，又花了2个小时找原因

    原来是因为没有默认网关
    这时，你只需设置一个默认路由即可，例如：

    route add -net 0.0.0.0 dev ppp0

    route -n
    如图

    再次输入IP111.cn检查成功
